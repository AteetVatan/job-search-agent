# Variant: Project

Leads with a specific AI project that matches the job description domain. Best when the JD mentions healthcare AI, LLM evaluation, geospatial, voice AI, or video generation.

---

## How to Use

Identify which project from `cv-source-of-truth.md` best matches the JD domain. Then use the template below, filling in the matching project details.

---

Dear Hiring Team,

I noticed the [JOB TITLE] role at [COMPANY NAME] focuses on [DOMAIN FROM JD], which is exactly what I built with [PROJECT NAME FROM cv-source-of-truth.md].

[INSERT 2 TO 3 SENTENCES ABOUT THE MATCHING PROJECT, COPIED DIRECTLY FROM cv-source-of-truth.md. DO NOT ADD DETAILS NOT IN THE CV.]

[INSERT 1 TO 2 SENTENCES ABOUT YOUR BROADER CURRENT WORK AND TOTAL EXPERIENCE, FROM cv-source-of-truth.md.]

[INSERT 1 SENTENCE ABOUT LOCATION, CITIZENSHIP, AND AVAILABILITY, FROM config.private.yaml → applicant section]

Best regards,
[NAME from config.private.yaml → applicant.name]
[EMAIL from config.private.yaml → applicant.email] | [PHONE from config.private.yaml → applicant.phone]
[PORTFOLIO from config.private.yaml → applicant.portfolio]
